# 團隊等級跨部門專案管理系統藍圖

## 1. 目標與設計原則

此系統的核心目標是把「任務執行」與「策略目標（OKR/KPI）」串成同一條可追蹤鏈路，讓個人、團隊與跨部門都能使用一致語言協作。

設計原則：

1. **一個真實來源（Single Source of Truth）**：任務、里程碑、目標、指標都在同一平台管理。
2. **雙向可追溯（Traceability）**：可由 OKR/KPI 往下看到專案與任務，也可由任務往上看到對應目標。
3. **角色分層可見（Role-based Visibility）**：個人看今日重點，主管看風險與預測，跨部門看依賴與交付。
4. **節奏化治理（Operating Rhythm）**：以日/週/月節奏推進，不只記錄現況，還要驅動行動。

## 2. 資訊模型（你要管理的 6 個物件）

1. **Objective（目標）**
   - 範例：提升企業客戶續約率。
2. **Key Result / KPI（關鍵結果）**
   - 範例：續約率由 78% 提升至 85%。
3. **Initiative / Program（策略方案）**
   - 為達成 KR 的跨專案方案。
4. **Project（專案）**
   - 有範圍、里程碑、時間與責任人。
5. **Milestone（里程碑）**
   - 專案關鍵節點。
6. **Task（任務）**
   - 最小可執行單位，可分派到個人。

建議關聯結構：

- `Objective 1:N Key Result`
- `Key Result 1:N Initiative`
- `Initiative 1:N Project`
- `Project 1:N Milestone`
- `Milestone 1:N Task`
- `Task N:1 Owner（人員）`

## 3. 你需要的 3 個視角（對應你的需求）

### A. 個人視角（你自己的任務與專案）

必備看板：

- **My Tasks（本週）**：按優先級、到期日、阻塞狀態排序。
- **My Projects（本月）**：顯示各專案完成率、風險與下一里程碑。
- **My Goal Impact**：你的任務對應到哪些 KR，貢獻分數如何。

建議欄位：

- 任務狀態（Todo / Doing / Blocked / Done）
- 預估工時 vs 實際工時
- 風險等級
- 依賴關係（前置任務）
- 關聯 KR / KPI

### B. 團隊視角（團隊成員專案進度）

必備看板：

- **Team Capacity Board**：各成員負載、閒置、超載預警。
- **Team Project Health**：專案健康度（綠/黃/紅）與風險原因。
- **Delivery Forecast**：里程碑準時達成機率。

建議機制：

- 每週固定更新「完成、下一步、風險、需協助」。
- 自動通知逾期任務與延遲里程碑。
- 主管只看例外（exception-based management），降低會議成本。

### C. 跨部門視角（跨部門協作專案）

必備看板：

- **Cross-functional Dependency Map**：部門間依賴與阻塞點。
- **Shared Milestone Tracker**：共同里程碑與 owner。
- **SLA / Hand-off Monitor**：需求交接與回應時效。

治理建議：

- 每個跨部門專案設 **DRI（直接責任人）** 與 **Steering Owner（決策責任人）**。
- 明確定義「升級規則（Escalation Path）」：阻塞 >48 小時自動升級。

## 4. OKR / KPI 串接方法（最重要）

每個任務至少要回答三件事：

1. 這個任務支援哪個 KR/KPI？
2. 影響程度是什麼（高/中/低）？
3. 完成後能產生哪些可量化變化？

建議在系統中強制欄位：

- `linked_objective_id`
- `linked_kr_id`
- `impact_score`（1-5）
- `metric_before`
- `metric_target`
- `metric_current`

儀表板 KPI 可包含：

- KR 覆蓋率：有對應任務的 KR 比例
- KR 推進率：目前值 / 目標值
- 目標貢獻分佈：各部門對 KR 的貢獻占比
- 風險燃盡圖：高風險項目數量隨時間變化

## 5. 推行節奏（90 天落地計畫）

### Phase 1（第 1-30 天）：定義與對齊

- 統一名詞、狀態定義、優先級規則。
- 整理現有專案與 KPI，建立 mapping。
- 先選 1 個團隊 + 1 個跨部門專案試點。

### Phase 2（第 31-60 天）：上線與自動化

- 導入標準模板：專案章程、里程碑模板、週報模板。
- 建立即時通知：逾期、阻塞、依賴延遲。
- 啟用主管儀表板與跨部門儀表板。

### Phase 3（第 61-90 天）：治理與優化

- 週會專注「風險/決策」，不重複報數據。
- 月度回顧檢查 KR 推進與資源配置。
- 季度調整優先級，關閉低價值專案。

## 6. 會議與回報節奏（建議）

- **Daily（15 分）**：任務阻塞與今日重點。
- **Weekly（60 分）**：里程碑、風險、跨部門依賴。
- **Bi-weekly（45 分）**：OKR/KPI 變化與策略調整。
- **Monthly（90 分）**：管理層檢視投資報酬與資源重配。

## 7. 成功衡量指標（系統是否有效）

1. 里程碑準時率（On-time Milestone Rate）
2. 跨部門阻塞平均解決時間
3. KR 達成率與預測準確度
4. 會議時間下降比例
5. 任務重新開啟率（品質指標）

## 8. 可立即開始的最小版本（MVP）

若你要最快啟動，可先做以下 5 件事：

1. 建立「Objective → KR → Project → Task」基本欄位。
2. 所有任務必填 KR 關聯與截止日。
3. 啟用個人/團隊/跨部門三個儀表板。
4. 每週固定產出風險清單與處置 owner。
5. 每兩週檢討「做了哪些事」改為「KR 前進了多少」。
